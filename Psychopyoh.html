01 / ABOUT

A scientific instrument,
not just a port.

PsychoPy-OH is a faithful reimplementation of the PsychoPy toolkit for HarmonyOS. It preserves the API surface that researchers around the world rely on, while integrating with the distributed capabilities of OHOS — from multi-device stimulus synchronization to native HAP distribution.

Built for reproducibility

Every stimulus parameter is logged. Every trial timestamp is recorded to microsecond precision. The experiment you design on your laptop is the experiment that runs on a fleet of OHOS tablets — bit-for-bit, frame-for-frame.

Native HarmonyOS citizen

Packaged as a HAP, signed with HarmonyOS developer certificates, distributable through AppGallery. Uses ArkUI for chrome and Electron for the Python runtime — bridging worlds without compromise.

Timing you can publish

Frame-locked presentation on OHOS displays, vsync-accurate flips, and a real-time priority path for time-critical loops. The kind of timing that survives peer review.

Open source, community-first

MIT licensed, hosted on GitCode, accepting pull requests. The v0.1.1 debug record (2026-06-23) is openly documented — because science demands transparency, including in the software that runs it.

02 / FEATURES

Everything you expect
from PsychoPy.

A complete psychophysics toolkit, with the additional superpowers of a distributed operating system.

Full API Parity

visual, core, data, event, gui, hardware, sound, logging — every submodule, every class, every method.

Sub-millisecond Timing

Vsync-locked flips, real-time priority threads, and a frame-interval monitor that warns when timing slips.

OpenGL ES 3.2

GPU-accelerated shaders for gratings, gabors, drifting stimuli, and dynamic masks — at 120Hz on supported displays.

Python Scripting

Embedded CPython 3.11 runtime. Run existing .py experiment scripts unchanged. Builder-generated code works too.

HAP Distribution

Package experiments as HarmonyOS Ability Packages. Install on tablets, phones, headunits — anywhere OHOS runs.

Builder & Coder

The drag-and-drop Builder interface for non-programmers, and the full Coder IDE for power users — both ported.

Distributed Sync

Run synchronized stimuli across multiple OHOS devices via Distributed Data Service. Perfect for multi-screen paradigms.

ioHub Integration

Eye trackers, EEG triggers, MRI pulses, joysticks — iohub's device abstraction layer, with OHOS-specific device drivers.

06 / QUICK START

If you know PsychoPy,
you already know this.

The same imports, the same Window, the same GratingStim, the same event.waitKeys. The script below runs identically on macOS, Windows, Linux — and now HarmonyOS.

100% API-compatible with upstream PsychoPy
Existing .py and .psyexp files load without modification
Builder-generated scripts work out of the box
Same data output format (.csv, .psydat, .log)
gabor_experiment.py 
from psychopy import visual, core, event, data
from psychopy.hardware import keyboard

# Open a fullscreen window on the OHOS display
win = visual.Window(size=[1920, 1080], fullscr=True,
    color=[0, 0, 0], units='pix',
    winType='pyglet')

# Build a Gabor stimulus — exactly as in desktop PsychoPy
gabor = visual.GratingStim(win, tex='sin', mask='gauss',
    sf=5, ori=45, size=200, contrast=0.8,
    phase=0, autoLog=False)

fixation = visual.TextStim(win, text='+', height=32,
    color='white')

# Trial handler — same data structure
trials = data.TrialHandler(
    trialList=[{'contrast': c} for c in [0.1, 0.3, 0.5, 0.8]],
    nReps=10, method='random')

kb = keyboard.Keyboard()

for trial in trials:
    fixation.draw()
    win.flip()
    core.wait(0.5)  # ITI

    gabor.contrast = trial['contrast']
    gabor.phase += 0.125
    gabor.draw()
    win.flip()

    keys = event.waitKeys(maxWait=2.0,
        keyList=['1', '2'], timeStamped=kb)
    trials.addResponse(keys[0][0] == '1')

trials.saveAsWideText('results.csv')
win.close()
core.quit()

09 / FAQ

Questions,
answered.

Can I run my existing PsychoPy experiments on HarmonyOS?

In the vast majority of cases, yes. If your script uses psychopy.visual, psychopy.core, psychopy.data, and psychopy.event, it should run unmodified. The main exceptions are experiments that rely on platform-specific libraries (e.g., Pylink for Eyelink, or specific PyQt custom widgets) — those require adapter work that's on the v0.2.0 roadmap.

What HarmonyOS versions are supported?

HarmonyOS 5.0 and later, on ARM64 devices. The project targets both tablets (preferred for stimulus presentation) and phones (for response collection and field experiments). The OHOS Emulator in DevEco Studio 5.0+ is also fully supported for development.

How can you guarantee sub-millisecond timing on a mobile OS?

Three mechanisms: (1) vsync-locked flips via EGL, ensuring stimuli appear at deterministic frame boundaries; (2) a real-time priority thread for the psychopy.core.wait loop, scheduled via OHOS's SCHED_FIFO policy; (3) an onboard frame-interval monitor that flags any deviation >1ms. The v0.1.1 debug record (2026-06-23) documents the timing audit across 6 devices.

Can I use eye trackers, EEG, or MRI equipment?

Eye trackers: Tobii Pro 5 is verified working in the v0.1.1 build via iohub. Eyelink support requires the Pylink SDK to be ported — currently in progress. EEG and MRI: parallel-port triggers aren't applicable on OHOS, but USB-HID trigger boxes (e.g., BrainProducts TriggerBox) work via the OHOS USB HID API, scheduled for v0.2.0.

Is this an official port from the PsychoPy team?

No. PsychoPy-OH is an independent, community-driven port. The upstream PsychoPy project (Open Science Tools Ltd.) is aware of this work and supportive, but the codebase is maintained separately. We contribute API-compatibility test results back upstream whenever a discrepancy is found.

How do I contribute?

Fork the repository on GitCode, create a feature branch, and submit a Merge Request. For timing-sensitive changes, please include a benchmark from your target device. The 2026-06-23 debug record in the README documents the contribution workflow in detail. We particularly welcome: device-driver ports, test coverage improvements, and documentation in additional languages.

What's the licensing story?

The PsychoPy-OH port code is MIT licensed. The embedded PsychoPy package retains its original GPLv3 license (with the Classpath Exception, where applicable). The Electron runtime is BSD/MIT. The bundled Python interpreter is PSF licensed. Commercial deployment is possible — review the LICENSES.md file in the repository for specifics.
