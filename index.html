01 / ABOUT

A scientific instrument,
not just a port.

PsychoPy-OH is a faithful reimplementation of the PsychoPy toolkit for HarmonyOS. It preserves the API surface that researchers around the world rely on, while integrating with the distributed capabilities of OHOS.

Built for reproducibility

Every stimulus parameter is logged. Every trial timestamp is recorded to microsecond precision. The experiment you design on your laptop is the experiment that runs on a fleet of OHOS tablets — bit-for-bit, frame-for-frame.

Native HarmonyOS citizen

Packaged as a HAP, signed with HarmonyOS developer certificates, distributable through AppGallery. Uses ArkUI for chrome and Electron for the Python runtime.

Timing you can publish

Frame-locked presentation on OHOS displays, vsync-accurate flips, and a real-time priority path for time-critical loops.

Open source, community-first

Apache 2.0 (template) + GPLv3 (PsychoPy upstream), hosted on GitCode and GitHub. The v0.1.2 debug record (2026-06-23) is openly documented.

08 / FAQ

Questions,
answered.

Can I run my existing PsychoPy experiments on HarmonyOS?

In the vast majority of cases, yes. If your script uses psychopy.visual, psychopy.core, psychopy.data, and psychopy.event, it should run unmodified.

What HarmonyOS versions are supported?

HarmonyOS 5.0 and later, on ARM64 devices. Tablets preferred for stimulus presentation, phones for response collection.

How can you guarantee sub-millisecond timing on a mobile OS?

Three mechanisms: (1) vsync-locked flips via EGL; (2) real-time priority thread via SCHED_FIFO; (3) onboard frame-interval monitor flagging >1ms deviations.

Can I use this for ERP / EEG studies?

Yes. Trigger codes are sent via USB-HID on OHOS, synchronized to screen flips via win.callOnFlip() with measured jitter <1ms. Sufficient for N400, P300, and MMN.

What eye trackers are supported on HarmonyOS?

Tobii Pro 5 is verified working in v0.1.2 via OHOS USB driver and ioHub. Eyelink support requires porting Pylink SDK — in progress for v0.2.0.

How accurate is the stimulus timing for psychophysics?

On 60Hz displays: 16.6ms ±1ms per frame. On 120Hz: 8.3ms ±0.5ms. Frame-interval monitor logs every flip; deviations >2ms are flagged.

Is this an official port from the PsychoPy team?

No. PsychoPy-OH is an independent, community-driven port by A9iska. Upstream PsychoPy (Open Science Tools Ltd.) is aware and supportive, but the codebase is maintained separately.

How do I contribute?

Fork on GitCode or GitHub, create a feature branch, submit a Merge Request. For timing-sensitive changes, include a benchmark. We welcome device-driver ports, test coverage, and multilingual docs.

Why are icons not displaying after build?为什么构建后图标不显示？

Root cause: The dist/ directory was copied from another project or git branch instead of being rebuilt. Compiled SvelteKit JS chunks contain icon paths specific to the source version. Stale dist/ produces broken <img> / <svg><use> references.

Fix:

$ cd web_engine/src/main/resources/resfile/resources/app/
$ npx vite build
# Then rebuild HAR + HAP
$ hvigorw clean --no-daemon
$ hvigorw --mode module -p module=web_engine@default assembleHar
$ hvigorw --mode module -p module=electron@default assembleHap -p buildMode=debug

Lesson: dist/ is never tracked in git; must be generated locally via npx vite build. Never copy dist/ manually.

The technology stack, layer by layer.

Electron 28

CHROMIUM 120 · NODE.JS 20

The desktop application framework that hosts both the PsychoPy Builder/Coder UI and the embedded Python runtime. Cross-compiled for arm64-linux-ohos with custom patches for OHOS window management and EGL context handling.

CPython 3.11

EMBEDDED · STATIC BUILD

The complete CPython interpreter, statically linked and embedded within the HAP. Ships with pip, numpy, scipy, Pillow, and the full psychopy package.

OpenGL ES 3.2

GPU SHADERS · VSYNC LOCKED

The rendering backend for all psychopy.visual stimuli. Custom GLSL shaders implement sinusoidal gratings, Gaussian masks, and drifting textures. Frame-locked to the display's vsync.

HarmonyOS 5.0+

ARKUI · DDS · USB-HID

The native platform layer. ArkUI provides the system chrome and lifecycle management. Distributed Data Service enables multi-device stimulus sync. USB-HID API connects to EEG trigger boxes and eye trackers.

Git LFS sync critical for native libsGit LFS 同步对原生库至关重要

Problem: libelectron.so (160MB) is tracked via Git LFS. Cloning without LFS auth leaves a 137‑byte pointer file, causing SIGTRAP crashes.

Fix: Ensure git lfs pull completes before building. The binary is cached at .git/lfs/objects/.../013974... after a successful pull.

Verify libs/arm64-v8a/libelectron.so is 160MB before building.

ISSUE #001

EGL context-loss race

When the OHOS compositor reclaimed the EGL surface during background transitions, the Electron GPU process crashed silently. Fixed by wrapping eglSwapBuffers in a recovery loop with context recreation.

SEVERITY: Critical · FIXED: v0.1.2
ISSUE #002

Tobii Pro 5 handshake

The iohub eye-tracker handshake failed on first connect, requiring a manual restart. Root cause: OHOS USB permission dialog timing. Fixed by adding a 500ms retry delay after USB device enumeration.

SEVERITY: High · FIXED: v0.1.2
ISSUE #003

Frame timing jitter

On HarmonyOS 5.0.0 tablets, frame intervals showed occasional 33ms spikes. Root cause: background thermal throttling. Mitigated by requesting SCHED_FIFO priority and disabling CPU frequency scaling during experiments.

SEVERITY: Medium · FIXED: v0.1.2
D / DEVELOPMENT GUIDE

Building from source.

environment_setup.sh
# Prerequisites
$ sudo apt install build-essential cmake ninja-build
$ pip install meson pytest
# Clone with submodules
$ git clone --recursive https://gitcode.com/A9iska/psychopy-oh.git
$ cd psychopy-oh
# Download OHOS NDK (arm64-linux-ohos)
$ ./scripts/fetch-ndk.sh --version=5.0
# Build CPython for OHOS
$ cd cpython-build
$ ./configure --host=aarch64-linux-ohos --enable-shared --with-ensurepip=install
$ make -j$(nproc) && make install
# Bundle Python + psychopy into HAP resources
$ cd ../electron_runtime
$ python bundle.py --python-dir=../cpython-build/install --target=arm64-linux-ohos --include=psychopy,numpy,scipy,Pillow

01 Environment

Ubuntu 22.04+ or macOS 14+. 16GB RAM recommended for Electron cross-compilation. OHOS NDK 5.0+ required.

02 CPython Build

CPython is cross-compiled for arm64-linux-ohos using the OHOS NDK toolchain. The resulting libpython3.11.so is bundled into the HAP.

03 Electron Patch

Electron 28 is patched with 3 custom commits for OHOS: EGL surface recovery, USB-HID enumeration, and ArkUI lifecycle integration.

04 HAP Assembly

The final HAP is assembled by hvigorw, combining the Electron binary, Python bundle, ArkUI shell, and psychopy resources into a signed .hap file.

05 Testing

Run python -m pytest tests/ for unit tests, and ./scripts/timing-audit.sh for frame-precision benchmarks on a connected device.

Documentation

Translate the README, write tutorials, or document OHOS-specific quirks. We welcome contributions in Chinese, Japanese, Korean, and European languages.

Ready to contribute?
